I Know Nothing About College Basketball, But Heres The Bracket Prediction I Made Using Machine Learning

2021-03-23

I want to preface this with saying that this is the first time I have ever filled out a March Madness bracket. The extent of my college basketball knowledge before my Sports Data class was (1) my dad’s undying support for all of USC’s sports teams and (2) the tragedy that is Nebraska basketball–and neither got me very far.

So, when I was prompted to create a bracket using machine learning I had nothing to lose and everything to gain. I decided to blindly follow my model’s predictions, which honestly most likely got me farther than my limited knowledge could have.

Now we get into the fun stuff, how me–who knows nothing about basketball–decided to pick and choose statistics to predict the outcomes of tournament games.

Feature Engineering – AKA What I Used to Predict the Games

The data that I use to make my predictions is stolen from my professor, who scraped Sports Reference for every single Division I college basketball game starting with the 2014-2015 season.

Now, I get to try and explain what I used to predict these games.

My thinking process was that if I could figure out how effective a team was during a game, I could use that as the biggest chunk of my prediction.

So, I started with the number of possessions and points per possession, a good estimate on how efficient a team was.

Then, from the advice of a friend that knows a lot more about basketball than I do, I decided to use a Rolling Mean to only take the statistics from a team’s past 3 games. This decision was based off the thought that if a team was on a hot streak or a losing streak, that would be taken into consideration going into the tournament.

After running my model once with just those statistics and seeing Colgate find it’s way into my National Championship, I figured something else needed to be done.

Enter WinQuality–something I also borrowed from my professor–which uses a score margin and the opponents simple rating to see the quality of the win. Essentially if a team has a large amount of wins but they’re all against “bad” teams, I want to differentiate that from teams who beat “good” teams, hence the Colgate situation.

games <- read_csv("~/Downloads/cbblogs1521 (6).csv") %>%
  mutate(
  Possessions = .5*(TeamFGA - TeamOffRebounds + TeamTurnovers + (.475 * TeamFTA)) + .5*(OpponentFGA - OpponentOffRebounds + OpponentTurnovers + (.475 * OpponentFTA)),
  PPP = TeamScore/Possessions) %>%
  group_by(Team, Season) %>%
  mutate(
  Rolling_Mean_Possessions = rollmean(Possessions,k = 4, fill=Possessions),
  Rolling_Mean_PPP = rollmean(PPP, k = 4, fill=PPP),
  ScoreDifference = TeamScore - OpponentScore,
  WinQuality = case_when(is.na(OpponentSRS)== TRUE ~ ScoreDifference, TRUE ~ ScoreDifference + OpponentSRS),
) %>% ungroup() %>% 
  mutate(
    Location = case_when(
    str_trim(HomeAway) == "@" ~ "A",
    str_trim(HomeAway) == "N" ~ "N",
    TRUE ~ "H"
  ),
 Outcome = case_when(
  grepl("W", W_L) ~ "W", 
  grepl("L", W_L) ~ "L"
 )
) %>%
  mutate(Outcome = as.factor(Outcome))

To solve the problem of each team being in the data set twice (once as the Team and once as the Opponent), I used a joining function to force them into one data set with one set of statistics.

selectedgames <- games %>% select(Season, Team, Date, Opponent, Outcome, Rolling_Mean_Possessions, Rolling_Mean_PPP, TeamTotalRebounds, TeamTurnovers, TeamSRS, TeamSOS, WinQuality)
opponentgames <- selectedgames %>% select(-Opponent) %>% rename(Opponent = Team, Opponent_Rolling_Mean_Possessions = Rolling_Mean_Possessions, Opponent_Rolling_Mean_PPP = Rolling_Mean_PPP, OpponentTurnovers = TeamTurnovers, OpponentSRS = TeamSRS, OpponentSOS = TeamSOS, OpponentTotalRebounds = TeamTotalRebounds, OpponentWinQuality = WinQuality)
bothsides <- selectedgames %>% left_join(opponentgames, by=c("Opponent", "Date", "Season")) %>% na.omit() %>% select(-Outcome.y) %>% rename(Outcome = Outcome.x)

Lets Create the Model!

The first step is splitting my data into training and testing sets.

log_split <- initial_split(bothsides, prop = .8)
log_train <- training(log_split)
log_test <- testing(log_split)

The second step is creating a recipe for my data, which essentially just separates my ID data from my predicting data.

log_recipe <- 
  recipe(Outcome ~ ., data = log_train) %>% 
  update_role(Team, Opponent, Date, Season, new_role = "ID") %>%
  step_normalize(all_predictors())

log_mod <- 
  rand_forest() %>% 
  set_engine("ranger") %>%
  set_mode("classification")

The fourth step is to create a workflow using my recipe.

log_workflow <- 
  workflow() %>% 
  add_model(log_mod) %>% 
  add_recipe(log_recipe)

The fifth and final step is to fit my model to my training data.

log_fit <- 
  log_workflow %>% 
  fit(data = log_train)
